Errors

Common Error Types

Reference Error

A ReferenceError is thrown when the variable you are referencing does not exist (within the current scope) or it has been misspelled.

The at script.js:4:13 shows you where the error occurred, the line, and the character number.

Reference Error

A SyntaxError occurs when the code you are trying to run is not written correctly.

Type Error

A TypeError is thrown when:

  • an operand or argument passed to a function is incompatible with the type expected by that operator or function.
  • when attempting to modify a value that cannot be changed.
  • when attempting to use a value in an inappropriate way.

Stack Trace

Here is some script:

    const a = 5;
    const b = 10;
    
    function add() {
        return c;
    }
    
    function print() {
        add();
    }
    
    print();

If you open the console by pressing Fn + F12 you will see the following:

    Uncaught ReferenceError: c is not defined
        at add (script.js:5)
        at print (script.js:9)
        at script.js.12

The stack trace tells us:

  • c is not defined in scope of add(), which is declared on line 5
  • add() was called by print(), which was declared on line 9.
  • print() itself was called on line 12.

Function Declarations versus Function Expressions

Function Declaration Function Expression
Hoisted Not hoisted
Use function keyword followed by a name Often anonymous but also can be named
Stand alone as statements Part of a larger expression
Best for defining reusable stand-alone functions More flexible for dynamic use-cases: callbacks, IIFEs,etc.
